Why Cereblix is different

🧬Self-mutating algorithm

Every 4096 blocks NeuroMorph rebuilds its own semantics from chain entropy: opcode weights, program length, constants, AES keys all change. Designing frozen hardware for an algorithm that doesn't exist yet is impossible. This is lifelong ASIC resistance - not a promise, but a property of the design.

⚖️1 CPU = 1 vote

Random programs with random branches are death for GPUs: their thousands of cores diverge on the first branch and stall. Any laptop on Intel or AMD can mine. No farms - the network belongs to people, not warehouses of hardware.

🤝Fair 2026 launch

Zero premine. Zero team allocation. Zero VC. The genesis block is empty; coins exist only from mining. The first block was mined the day this site went live - you are exactly zero behind.

🌱Background energy

NeuroMorph stresses the CPU cache and pipeline, not power draw: mining on 2-4 threads barely heats the machine and lives on idle cycles. A coin of background computation, not megawatt halls.

📡Lightweight node

A full node is a single dependency-free binary (pure Go, stdlib). It runs on a $3 VPS, a Raspberry Pi or a home PC. The chain is human-readable JSONL: you can audit it by eye.

🔑Your keys only

The web wallet signs transactions in your browser (ed25519); the private key never leaves your device. The same key works in the CLI wallet. No sign-up, no email, no KYC.

How NeuroMorph works

For each block candidate (nonce) the algorithm:

  1. Unfolds a random program (384-768 instructions) from the header hash for a virtual machine: integer math, IEEE-754 float, hardware AES rounds, data-dependent branches.
  2. Fills a 2 MiB scratchpad - sized for the CPU's L2/L3 cache. Random access to it makes cheap external memory useless.
  3. Runs 32-64 execution loops, mixing registers back into memory - work cannot be skipped.
  4. Folds everything into SHA-256 and compares against the difficulty target.

And at every epoch boundary (4096 blocks ≈ 3 days) new VM rules are born from the chain hash. An ASIC shipped today executes a "different" algorithm one epoch later. The only device that doesn't care is a general-purpose processor.

Solo or pool?

🎯Solo

You mine straight against the network. Find a block and you keep the full 50 CRB - but it's a lottery: on a small CPU you might wait hours or days between wins. Best if you have a big multi-core rig.

cereblix-miner -addr crb1YOUR_ADDRESS

👥Pool

You mine together with everyone else. You earn a steady trickle proportional to your work, and the pool auto-pays your address once you reach 0.05 CRB (1% fee). Best for laptops and ordinary CPUs - far less variance.

cereblix-miner -addr crb1YOUR_ADDRESS -node https://cereblix.com/pool/api
💡 On the pool your miner logs "share accepted" - those are shares (proofs of work at an easier target), not full blocks. Your real reward lands as automatic pool payouts to your address. Same miner either way - only the -node changes.

FAQ

Why is "ASIC-resistant forever" true, not marketing?

Classic "ASIC-resistant" algorithms (Ethash, Equihash) were static - and chips were eventually built for them. NeuroMorph is never static: the execution rules are reborn each epoch from a chain hash that cannot be predicted. Any fixed hardware is obsolete within ~3 days. A device that can execute arbitrary future programs with branches, float and AES - that is a CPU. So an "ASIC for Cereblix" is an oxymoron: its silicon has to become a processor.

What about GPUs and botnets?

GPUs: random branches every few instructions cause warp divergence - the card runs at the speed of one or two of its scalar cores, i.e. slower than a laptop. Botnets are a problem for all CPU coins; our answer is economic: a low cap at launch makes them unprofitable, and egalitarian mining distributes the coin wider than any "fair" GPU launch.

Is this a fork of Monero/RandomX?

No. The codebase is written from scratch in Go (core + node + miner ~2500 lines, zero external dependencies). NeuroMorph conceptually extends RandomX's random-program idea, but adds the key part: epoch mutation of the VM semantics itself. RandomX is frozen forever - NeuroMorph is not frozen by principle.

How can I support the network?

Mine (even 1-2 threads helps security), run a node, mirror the source. The code is open: github.com/Cerebra-CBR/cereblix.

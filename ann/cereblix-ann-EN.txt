[size=12pt][b]🧠 In one line[/b][/size]
Cereblix (CRB) is a CPU-only proof-of-work coin on [b]NeuroMorph[/b] - a virtual machine that rewrites its own rules every epoch, so no GPU or ASIC can ever gain an edge. Go, zero dependencies, MIT, fair launch. It's [b]crypto for people, not farms[/b] - a serious, long-term project built to grow with its community and become genuinely useful.

[size=12pt][b]⚙️ Why NeuroMorph[/b][/size]
[list]
[li][b]Self-mutating[/b] - every 4096 blocks (~2.8 days) it rebuilds opcodes, program length, constants and AES keys from chain entropy. Hardware for an algorithm that doesn't exist yet is impossible.[/li]
[li][b]GPU-resistant[/b] - random data-dependent branches cause warp divergence; GPUs stall, a laptop competes.[/li]
[li][b]Farm-resistant[/b] - a 64 MiB memory-hard dataset prices out cheap cloud parallelism.[/li]
[li][b]Verifiable[/b] - checking a hash is cheap; producing one needs a real CPU.[/li]
[/list]

[size=14pt][b]❓ Cereblix FAQ[/b][/size]

[b]Is this a scam?[/b]
Verify, don't trust. Empty genesis = zero premine (check the explorer), 100% open source (MIT), chain and wallet inspectable, no sale of any kind. Everything was mined publicly.

[b]Where is the whitepaper?[/b]
The spec is ARCHITECTURE.md on GitHub (consensus, NeuroMorph VM, mutation, memory-hardness, networking) plus the open code. We chose a verifiable spec over a marketing PDF.

[b]How does the mutation work?[/b]
Every 4096 blocks the PoW VM re-derives its own rules (opcodes, program length, constants, AES keys) from an unpredictable chain hash. Each nonce runs a fresh random program over a 2 MiB scratchpad + 64 MiB dataset. Verifying is cheap; producing needs a real CPU. Any fixed hardware is obsolete within ~3 days.

[b]Why CPU-only?[/b]
1 CPU = 1 vote. GPUs stall on branchy random programs; the memory-hard dataset prices out cloud farms. A laptop or phone competes on equal footing - no ASICs, no farms.

[b]Premine or dev fund?[/b]
Zero. Empty genesis, no presale, no ICO, no team allocation. Confirm it on the explorer.

[b]How do I start with no coins?[/b]
Faucet (cereblix.com/faucet.html) for a little to try the wallet, then mine in the browser (cereblix.com/mine.html) or with the native miner. Pool is best for laptops.

[b]Pool or solo?[/b]
Solo = whole blocks, rarely (lottery). Pool = steady payouts proportional to your work. Same miner, just change -node. Each pool share is cryptographically bound to your address.

[b]Antivirus flags the miner?[/b]
False positive (PUA) - nearly all unsigned CPU miners get flagged. Add an exclusion or build from source.

[b]Is it fully decentralized? What is the "checkpoint"?[/b]
Normal decentralized PoW. During bootstrap there is an optional, removable checkpoint signed by the founder that only blocks deep reorg (51%) attacks while total hashrate is small - it cannot mint coins or censor transactions, and switches off as independent hashrate and nodes grow. Mining itself is fully 1 CPU = 1 vote.

[b]Listings? Price?[/b]
No timeline. Listings and liquidity follow real adoption, not announcements.

[b]How can I help?[/b]
Run a node, mine, build a pool/explorer/tool, write guides, or join Telegram (t.me/cereblix) and X (x.com/Cereblix). PRs welcome.
